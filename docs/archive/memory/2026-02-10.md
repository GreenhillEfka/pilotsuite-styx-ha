## Token-Optimierung / "Sparpläne"
User fragte nach unseren Token-Sparplänen → Erinnerung an Diskussion über Model-/API-Effizienz.

### Problem
- Heartbeat-Calls können teuer werden (alle ~30 min, großes Modell)
- Unnötig große Kontexte bei einfachen Routine-Checks
- Model-Overkill für simple Aufgaben

### Optimierungsstrategien (zur Ausarbeitung)
1. **Heartbeat-Effizienz**
   - Batch-Checks statt einzelne API-Calls
   - HEARTBEAT.md gezielter nutzen
   - Seltener checken bei wenig Aktivität

2. **Model-Routing** 
   - Kleinere/günstigere Modelle für Routine-Tasks
   - Großes Modell nur bei komplexen Anfragen
   
3. **Context-Management**
   - Minimale Kontexte für einfache Checks
   - Smart Memory-Loading (nur relevante Snippets)

4. **Cron vs Heartbeat**
   - Cron für zeitkritische/isolierte Tasks
   - Heartbeat für kontext-sensitive Batch-Checks
